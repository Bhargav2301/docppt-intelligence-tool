## 🚀 Overview

The **DocPPT Intelligence Tool** is a web-based PPTX intelligence platform that solves a common pain point for consultants, product managers, founders, and freelance builders: receiving AI-assisted slide decks that read like they were generated by a language model rather than written by a human presenter.

The tool accepts a `.pptx` upload, runs an explainable AI-likeness analysis on every text element across all slides, and rewrites flagged text into natural, concise, audience-ready language — while preserving the deck's structure, layout, non-text assets, and formatting with 100% fidelity.

All analysis and rewriting operates through **free, local, and small-model NLP pipelines** using deterministic feature engineering and optional local language model scoring rather than relying on paid external LLM APIs. A Gemini BYOK (Bring Your Own Key) option is also supported for higher-quality rewrites.

## 🛠️ Key Features

### 1. PPTX Text Extraction

- Extracts text slide-by-slide and shape-by-shape, keeping track of `slide_id`, `shape_id`, `paragraph_id`, and `run_id` for layout-safe round-trip reconstruction.
- Identifies the semantic role of each text block: title, subtitle, bullet, body, caption, table cell, or speaker note.
- Stores raw and normalized copies separately so analysis never corrupts source data.

### 2. Artifact Detection

Deterministic, rule-based detection of visible cleanup issues before style analysis:
- **Citation residue:** `[1]`, `[Source]`, `(Source)`, superscript-style references rendered as plain text.
- **Markdown leakage:** `**bold**`, `# heading`, backticks, raw list markers copied from markdown editors.
- **Smart-character and delimiter noise:** stray pipes, em-dash misuse, duplicated punctuation, malformed quotes.
- **Generation framing leftovers:** "Certainly,", "Here's a refined version:", "In conclusion,", "Overall,", "It is worth noting that".

Every auto-applied cleanup is logged for user review.

### 3. AI-Likeness Analyzer

Computes an explainable AI-likeness score for each text element using measurable linguistic signals across seven feature groups:

| Feature Group | Key Signals |
|---|---|
| Surface features | Sentence length variance, type-token ratio, punctuation density |
| Burstiness and rhythm | Sentence opener repetition, uniform bullet parallelism |
| Repetition signals | Tri-gram repeats, generic business phrase density |
| Lexical specificity | Abstract vs. concrete noun ratio, vague adjective share |
| Syntactic patterns | Passive voice rate, modifier stacking, compound structure overuse |
| Readability | Flesch-Kincaid grade, Gunning fog, discourse transition frequency |
| Optional model signals | Local language model perplexity, embedding anomaly distance |

Scores map to three bands: **low**, **moderate**, and **high** AI-likeness. Each result includes specific, human-readable reasons rather than a single opaque number. Short text elements (under 20 words) receive score dampening.

The tool never claims authorship certainty. It identifies **AI-like phrasing patterns** only.

### 4. Humanization Rewriter

A multi-stage controlled rewrite pipeline that diagnoses each text element's specific weaknesses and applies targeted corrections:

- **Planner:** only triggers generative rewriting for `moderate` or `high` AI-likeness segments. `low` segments are passed through with deterministic cleanup only — no LLM calls.
- **Rewriter:** generates multiple candidates from Gemini BYOK, managed endpoints, or local CPU/GPU models. Falls back to deterministic rule-based candidates if all model calls fail.
- **Judge:** rejects candidates that modify numbers or digits, introduce new named entities, contain emojis, use British English spellings, or exceed character constraints. Only meaning-preserving, layout-safe candidates pass.
- **Tone presets:** `presentation_concise`, `executive_polished`, `founder_clear`, `product_manager_direct`, `consulting_professional`.
- **Intensity levels:** Minimal (artifact cleanup only), Balanced (rhythm + phrasing), Strong (full compression + specificity transformation).

### 5. Layout-Safe PPTX Export

- Recompiles the deck with modified text runs at the exact source location (`slide_id`, `shape_id`, `paragraph_id`, `run_id`).
- Validates character count delta, line-break risk, and text-box overflow before inserting each rewrite.
- Classifies each change as `safe_replace`, `needs_shorter_option`, or `manual_review`.
- Guarantees 100% fidelity for all non-text assets: images, shapes, tables, charts, animations, and slide count.

### 6. Human-in-the-Loop Review UI

- **Per-segment AI-likeness Badges:** View the computed AI-likeness score with a distinct band label (low / moderate / high) and the top reason.
- **"Why flagged" Expandable Disclosures:** Expandable `<details>` disclosure panels showing `flag.explanation`, `flag.matched_text`, and `flag.recommendation` for each flag.
- **Safety Label Badges:** Surface layout overflow risks (`needs_shorter_option` / `⚠ Shorter needed` and `manual_review` / `🔴 Manual review`) directly on segment cards.
- **Side-by-side Diff View:** Direct side-by-side comparison and editing of original vs rewritten text with change highlighting.
- **Collapsible "Run Rewrite" Panel:** Direct tone and intensity controls on the review page to batch-rewrite all flagged segments.
- **Per-segment "Re-run" Button:** Targeted single-segment retries to iterate on individual sentences without reprocessing the full deck.
- **Slide-wide Progress Bar:** Horizontal progress indicator showing the portion/count of accepted, rejected, and pending segments.
- **Pre-export Safety Modal:** Alerts when `manual_review`-labeled segments are still accepted, showing a count before allowing export.
- **One-click Revert:** Revert to original text per segment.
- **Settings Persistence:** Tone preset, intensity, and local model settings persist via `/settings`.

## ⚡ Getting Started

### 📋 Prerequisites

- **Frontend:** Node.js v20+ and `npm`
- **Backend:** Python 3.10+ and `pip`
- **Database:** SQLite (local development default) or PostgreSQL (production)

---

### 🔧 1. Backend Setup (`services/nlp`)

```bash
cd services/nlp

# Create and activate virtual environment
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Start development server
uvicorn main:app --reload --port 8000
```

FastAPI docs: [http://localhost:8000/docs](http://localhost:8000/docs)

> **Note:** On first startup the server runs database migrations and seeds a default developer account:
> - **Username:** `local_user@example.com`
> - **Password:** `password`

---

### 💻 2. Frontend Setup (`apps/web`)

```bash
cd apps/web
npm install

# Create environment config
echo "NEXT_PUBLIC_API_BASE_URL=http://localhost:8000" > .env.local

# Start development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

> **Gemini BYOK (Bring Your Own Key):** To use Gemini for rewriting, enter your Gemini API key in Settings → Rewrite Preferences. The key is stored in `sessionStorage` only (cleared on tab close) and is encrypted with the backend's RSA public key before transit. If no key is provided, the tool uses the configured local model or falls back to deterministic rewrite rules.

## ⚠️ Known Constraints

- Input must be `.pptx` format only. `.docx`, Google Doc URLs, and raw text input are not supported.
- The AI-likeness score is a style-pattern signal, not an authorship claim. Short text elements (under 20 words) receive score dampening.
- Rewrites that exceed the estimated character limit for a text box are flagged as `needs_shorter_option` and routed back for a tighter candidate. Auto-apply is blocked until a safe rewrite is confirmed.
- Speaker notes are analyzed and rewritten separately from slide body text and are never inserted into slide shape runs.
- The Gemini BYOK key is stored in `sessionStorage` only and is not persisted across browser sessions.
