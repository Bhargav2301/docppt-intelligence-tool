# Export Formats Design

## Overview
Adds support for exporting document analysis results to PDF and Word (.docx).

## Structure
Both formats will include:
1. **Title**: Doc Analysis: {Session Name}
2. **Summary**: Structured summary of the product.
3. **Product Description**: Overview, Target Users, Problem, Solution.
4. **Requirements**: Grouped list of technical/business requirements.
5. **Assumptions**: Marked separately.

## Libraries
- **PDF**: `reportlab` or `fpdf2` (Local, no API needed).
- **Word**: `python-docx` (Local, no API needed).

## Styling
- Minimalist professional layout.
- Headers for sections.
- Bullet points for requirements.
- Footer with "Generated by DocPPT Intelligence Tool".

## Backend Integration
- `GET /api/doc/{id}/export/pdf`
- `GET /api/doc/{id}/export/docx`
- Files are generated on-the-fly or cached in the `files` table.
